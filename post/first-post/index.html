一、静态生成器选择

常见的选择：

  • Jekyll
  • Hexo
  • Hugo

最终选择Hugo。个人理由是：

  1. 生成速度快
  2. 安装简单
    • Hugo是一个编译好的二进制，不需要安装语言环境
  3. Hugo是Go语言写的，而我之前用过Go语言

二、Hugo使用

2.1 安装

macOS系统上安装：

1
2
3

brew install hugo

hugo version

2.2 创建新站点

1
2
3

cd <path>

hugo new site <blog-directory>

注：下文中的所有hugo命令都需要在<blog-directory>目录下执行。

2.3 添加主题

查看可选的主题：hugo themes。

比如，要使用even主题:

  1. 安装主题： 
    1
2
3
4
5

    cd <blog-directory>

git submodule add https://github.com/olOwOlo/hugo-theme-even themes/even

cp ~/<blog-directory>/themes/even/exampleSite/config.toml .
  2. (可选) 根据个人需要来修改主题配置 
    1

    vim config.toml

2.4 添加新文章

1

hugo new post/<post-name>.md

以上命令会：

  • 在<blog-directory>/content下创建目录及文件
  • 在<post-name>.md中添加一些文件头部信息 
    1
2
3
4
5

    ---
title: "First Post"
date: 2022-04-24T10:47:42+08:00
draft: true
---

注意：

  1. 如果是使用上面提到的even主题，那这里必须使用post/而不能是posts/
  2. draft为true的文件不会被部署

2.5 本地预览效果

  1. 启动本地服务器: 
    1

    hugo server -D
    • -D 表明draft为true的文件也要处理
  2. 在浏览器中进行查看。

2.5 生成静态页面

如果预览之后没有问题，就可以生成静态页面以便后面用来部署：

1

hugo -D
  • 默认生成的页面文件会放到./public目录下
  • 可以通过修改config.toml里的publishdir项来设置输出目录

3. 部署

3.1 GitHub Pages

创建GitHub Pages:

  1. 创建一个空的仓库：<username>.github.io
    • 仓库的名字必须以你github用户名开头
  2. 将这个仓库clone到本地
  3. 将hugo生成的静态页面拷贝到本地的仓库下
  4. 提交并push

完成以上四步之后，就可以通过地址 <username>.github.io 来访问你的个人网站。
